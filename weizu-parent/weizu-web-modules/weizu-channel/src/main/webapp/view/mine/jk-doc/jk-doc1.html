    二、接口列表

    1.提交充值订单

    • 请求方式:GET
    • 举例地址:http://120.55.162.224:8082/flowsys/weizuAPI/charge?userName=xxxx#ber=13333333333&flowsize=5&userOrderId=123123&sign=225345C671534666D1232ACE718A690B
    • 请求地址:http://120.55.162.224:8082/flowsys/weizuAPI/charge
    • 请求参数:
    参数名称 参数类型 必须 注释
    userName String 32 是 用户账号
    number String 11 是 手机号码
    pgSize int 是 充值流量大小 5M参数为5,10M参数为10,1G参数为1024,2G参数为2048以此类推
    billType int 否 值为1代表对公，值为0代表对私。不传0即默认为1
    scope int 否 值为0代表全国漫游流量，值为1代表省内流量，值为2代表省漫游流量。不传即默认为2
    userOrderId String 否 商户订单号，可选参数，最多支持40位的字符串
    sign String 是 MD5(userName=xxxx&userPass=xxxxx&apikey=8a982a8a5e9fd1c1015e9fd1c1900000)并大写

    2.充值结果回调

    • 请求方式:POST/GET
    • 回调地址:用户设置的或者订单递交的reporturl
    • GET方式:?tipCode=0&transaction_id=201504210952082292343482&user_order_id=201504210952123&number=13333333333&status=4
    • 响应内容: ok 为成功，其他则失败
    • 响应说明: 超时或者非200状态码的 http 响应系统会判断回调失败，并且重新发起回调，共四次回调，间隔为30秒、60秒、120秒，300秒
    参数名称 类型 注释
    tipCode int 6 数据操作错误码，0代表成功，非0代表异常
    transaction_id string 32 订单号
    user_order_id string 40 用户提交的订单号
    number string 11 手机号码
    status int 3 充值结果:成功4失败8

    3.获取账号余额

    • 请求方式:GET
    • 请求地址:http://139.224.70.161:32001/api/v1/getBalance?username=xxxx&sign=1B0D437C2FEE8E14D33E487AEA504DA2
    • 数据格式:key=value
    • 请求参数:
      参数名称 是否必须 注释
      username 必须 用户账号
      sign 必须 MD5(username=xxxx&apikey=098f6bcd4621d373cade4e832627b4f6)并大写
    • 返回JSON数据:
    1. {
    2.     "tipCode": 0,
    3.     "tipMsg": "success",
    4.     "balance": "20.05"
    5. }
    • 返回说明:
    参数名称 类型 注释
    balance string 余额，单位元

    4.获取某一订单详情

    • 请求方式:GET
    • 请求地址:http://139.224.70.161:32001/api/v1/orderState?username=xxxx&order_id=order_id&user_order_id=user_order_id&telphone=13333333333&sign=1B0D437C2FEE8E14D33E487AEA504DA2
    • 数据格式:key=value
    • 请求参数:
    参数名称 是否必须 注释
    username 必须 用户账号
    order_id 否 订单ID，即transaction_id（user_order_id 和 order_id 至少传一个值，同时传优先选择 order_id)
    user_order_id 否 商户订单ID（user_order_id 和 order_id 至少传一个值，同时传优先选择 order_id)
    telphone 否 13333333333
    sign 必须 MD5(username=xxxx&apikey=098f6bcd4621d373cade4e832627b4f6)并大写

    5.错误代码

    • 发送订单结果:
    tipCode tipMsg
    0 提交成功
    50000 服务器内部错误
    50005 number参数有误
    50006 flowsize参数有误
    50008 签名有误
    50012 参数有误
    50004 订单重复
    55006 订单不存在
    • 订单状态结果只有0:未充值，1:等待充值，2:正在充值，4:充值成功，8:充值失败
