    二、接口列表

    1.提交充值订单

    • 请求方式: GET
    • 举例地址:http://120.55.162.224:8082/flowsys/weizuAPI/charge?userName=xxxx&number=13333333333&pgSize=5&userOrderId=123123&sign=225345C671534666D1232ACE718A690B
    • 请求地址:http://120.55.162.224:8082/flowsys/weizuAPI/charge
    • 请求参数:
    参数名称 参数类型 必须 注释
    userName String 32 是 用户账号
    number String 11 是 手机号码
    pgSize int 是 充值流量大小 5M参数为5,10M参数为10,1G参数为1024,2G参数为2048以此类推
    billType int 否 值为1代表对公，值为0代表对私。不传0即默认为1
    scope int 否 值为0代表全国漫游流量，值为1代表省内流量，值为2代表省漫游流量。不传即默认为2
    userOrderId String 否 商户订单号，可选参数，最多支持40位的字符串
    sign String 是 MD5(userName=xxxx&userPass=xxxxx&apikey=8a982a8a5e9fd1c1015e9fd1c1900000)并大写

    * 返回JSON数据:

    1. {
    2.     "tipCode": 5001,
    3.     "tipMsg": "success",
    4.     "chargePo": {
    5.         "orderId": 745830224406515712,
    6.         "number": "13333333333",
    7.         "pgSize": 5
    8.         "billType": 1
    9.     }
    10. }
    • 返回说明:
    参数名称 类型 注释
    tipCode int tipCode为5001表示提交成功非5001提交异常(除错误代码表里的值可以直接失败，其余非5001请看作异常，不可直接失败)
    tipMsg String success表示充值提交成功
    chargePo object 订单对象
    orderId Long 订单号
    number String 11 手机号码
    pgSize int 流量大小
    billType int 充值账户类型：值为1代表对公账户，值为0代表对私账户。

    2.获取账号余额

    • 请求方式:GET
    • 请求地址:http://120.55.162.224:8082/flowsys/weizuAPI/my_balance?userName=xxxx&sign=1B0D437C2FEE8E14D33E487AEA504DA2&billType=0
    • 数据格式:key=value
    • 请求参数:
      参数名称 参数类型 必须 注释
      userName String 32 是 用户账号
      sign String 是 MD5(userName=xxxx&userPass=xxxxx&apikey=8a982a8a5e9fd1c1015e9fd1c1900000)并大写
      billType int 否 值为1代表对公，值为0代表对私。不传0即默认为1
    • 返回JSON数据:
    1. {
    2.     "tipCode": 5001,
    3.     "tipMsg": "用户验证成功",
    4.     "balance": "20.05"
    5.     "billType": 1
    6. }
    • 返回说明:
    参数名称 类型 注释
    accountBalance String 余额，单位元
    billType int 账户类型：值为1代表对公账户，值为0代表对私账户。

    3.获取某一订单详情

    • 请求方式:GET
    • 请求地址:http://120.55.162.224:8082/flowsys/weizuAPI/my_order_state?userName=xxxx&orderId=745830224406515712&sign=1B0D437C2FEE8E14D33E487AEA504DA2
    • 数据格式:key=value
    • 请求参数:
    参数名称 参数类型 必须 注释
    userName String 32 是 用户账号
    orderId Long 是 订单ID
    number String 11 否 13333333333
    sign String 是 MD5(userName=xxxx&userPass=xxxxx&apikey=8a982a8a5e9fd1c1015e9fd1c1900000)并大写

    * 返回JSON数据:

    1. {
    2.     "tipCode": 5001,
    3.     "tipMsg": "参数正确",
    4.     "OrderPo": {
    5.         "orderId": 745830224406515712,
    6.         "userOrderId": "123",
    7.         "number": "13333333333",
    8.         "pgSize": 10,
    9.         "chargeFee": "1.00",
    10.         "createdAt": "2017-10-17 14:50:33",
    11.         "status": 1,
    12.         "statusMSg": "充值成功"
    13.     }
    14. }
    • 返回说明:
    参数名称 类型 注释
    order object 订单对象
    orderId Long 订单号
    userOrderId String 32 商户订单号
    number String 11 手机号码
    pgSize String 16 充值流量大小
    chargeFee float 6,2 充值费用
    createdAt String 充值时间
    status int 订单状态，0:充值失败，1:充值成功，2:正在充值
    msg String 订单状态信息

    5.错误代码

    • 发送订单结果:
    tipCode tipMsg
    5001 提交成功
    5000 服务器内部错误
    5009 number参数有误
    5008 pgSize参数有误
    0 用户验证错误
    5504 订单号不存在
    • 订单状态结果只有 0:充值失败，1:充值成功，2:正在充值
